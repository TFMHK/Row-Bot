# ארכיטקטורת המערכת — סירה אוטונומית

> **שינוי ארכיטקטוני (2026-07-17):** בעקבות חוסר-יציבות בקישור הרדיו, לוגיקת
> **הניווט האוטונומי הועברה אל הארדואינו של הסירה ונצרבת עליו**. קישור הרדיו
> משמש כעת רק לתמונת שו"ב ב-UI, מעבר בין מצב אוטונומי לידני, ופקודות ידניות.
> ראו "מצבי שליטה" למטה. (הקישור בפועל הוא RH_ASK 433MHz חצי-דופלקס, לא RF24.)

## סקירה כללית

המערכת מורכבת משלושה רכיבים עיקריים:

```
┌─────────────────────────────────────────────────────────┐
│                      מחשב (PC)                          │
│  - תמונת שו"ב (תצוגת מכ"ם, טלמטריה)                     │
│  - מעבר בין מצב אוטונומי לידני + פקודות ידניות         │
│  - ניווט אוטונומי רץ כאן רק בסימולטור (mock) לבדיקות   │
└───────────────────────┬─────────────────────────────────┘
                        │ USB / Serial (CSV)
                        ▼
┌─────────────────────────────────────────────────────────┐
│            ארדואינו חוף (shore) — ממסר                  │
│  - מקבל פקודות מהמחשב, דוחס+חותם (MAC) ומשדר (RH_ASK)   │
│  - מקבל טלמטריה מהסירה, מאמת ומעביר למחשב                │
│  - אין לוגיקת ניווט — רק ממסר שקוף                      │
└───────────────────────┬─────────────────────────────────┘
                        │ RH_ASK 433MHz (חצי-דופלקס)
                        ▼
┌─────────────────────────────────────────────────────────┐
│            ארדואינו סירה (boat) — אוטונומי               │
│  - **מריץ את לוגיקת הניווט האוטונומי מקומית** (על-הסיפון)│
│  - קורא חיישני אולטרה-סוני (x4) בקצב מקומי מלא          │
│  - מפעיל מנועים (H-Bridge) לפי המצב (אוטונומי/ידני)     │
│  - שולח טלמטריה חזרה לחוף לתמונת שו"ב                    │
└─────────────────────────────────────────────────────────┘
```

## עיקרון מרכזי: ניווט אוטונומי על-הסיפון

קישור הרדיו איטי ולא-יציב, והחיישנים יושבים **על הסירה**. לכן לוגיקת ההימנעות
רצה על הארדואינו של הסירה, בקצב החיישנים המקומי המלא — לא מעל הקישור האיטי.
כך הניווט אינו תלוי ביציבות התקשורת.

- **המחשב** מספק תמונת שו"ב, מעבר-מצבים ופקודות ידניות בלבד.
- **הסירה** מקבלת החלטות ניווט (כיוון) בעצמה במצב אוטומטי, ומשתמשת ב**עוצמות
  המנוע שהחוף שולח** עם המעבר לאוטומטי (עוצמה מכוילת לכל צד — כיול נשאר בחוף).
- הניווט של קוד המחשב (app.js) עדיין רץ **בסימולטור (mock)** לצורכי בדיקה בלבד.

**⚠️ סיכון פעיל:** החיישן האחורי (usRadar) אינו עובד => אין חישה לאחור. לכן
הניווט האוטונומי המובנה **מעדיף סיבוב-במקום** לעבר הצד הפתוח יותר, וטווח ההימנעות
הקדמי הוגדל כדי להגיב מוקדם. יוצא-דופן: כשההתנגשות החזיתית קרובה ומסוכנת, הסירה
מבצעת **פרץ קצר של נסיעה לאחור** — סיכון אחורי לא-ודאי עדיף על התנגשות חזיתית ודאית.

## פרוטוקול תקשורת

### מחשב → ארדואינו חוף (Serial, 115200 baud)

```
<leftSpeed>,<rightSpeed>,<winchSpeed>,<radarAngle>\n
```

| שדה          | טווח        | תיאור                            |
| ------------ | ----------- | -------------------------------- |
| `leftSpeed`  | -255 עד 255 | מהירות מנוע שמאל (שלילי = אחורה) |
| `rightSpeed` | -255 עד 255 | מהירות מנוע ימין                 |
| `winchSpeed` | -255 עד 255 | מהירות כננת הרשת                 |
| `radarAngle` | 0 עד 180    | זווית סרוו המכ"ם                 |

### ארדואינו חוף → מחשב (Serial, 115200 baud)

```
<usRadar>,<usFront>,<usLeft>,<usRight>,<radarAngle>\n
```

| שדה       | יחידה | תיאור                     |
| --------- | ----- | ------------------------- |
| `usRadar` | ס"מ   | מרחק חיישן המכ"ם (מסתובב) |
| `usFront` | ס"מ   | מרחק חיישן קדמי קבוע      |
| `usLeft`  | ס"מ   | מרחק חיישן שמאלי קבוע     |
| `usRight` | ס"מ   | מרחק חיישן ימני קבוע      |
| `radarAngle` | מעלות | זווית הסרוו בזמן הדגימה |

## מצבי שליטה

המצב (ידני/אוטומטי) נשלח בכל פקודה (שדה `mode`) והסירה "נועלת" אותו. הסירה
**לעולם אינה יוזמת מצב אוטומטי בעצמה** — רק פקודת mode=1 מהחוף מפעילה אותו.

### 1. שליטה ידנית (Manual, mode=0)

- המשתמש שולט במנועים ישירות (מקלדת / ג'ויסטיק); הסירה מבצעת את הפקודה שהתקבלה
- תצוגת מכ"ם בזמן אמת על המסך (בנויה מנתוני `usRadar` + `radarAngle`)
- הטבעת ירוקה

### 2. שליטה אוטונומית (Autonomous, mode=1)

- **הסירה מנווטת בעצמה** על-הסיפון בקצב החיישנים המקומי; המחשב אינו נוהג
- המחשב ממשיך להציג תמונת מכ"ם מהטלמטריה ומאפשר להשתלט (מעבר לידני) בכל רגע
- הטבעת כחולה

### מעבר בין מצבים

- **המחשב בלבד** שולט במעבר בין מצב ידני לאוטומטי (מתג ב-UI => שדה `mode` בפקודה)

### התנהגות באובדן קשר (Failsafe)

| מצב בזמן הניתוק | התנהגות הסירה                                   | חיווי טבעת |
| -------------- | ---------------------------------------------- | ---------- |
| **אוטומטי**    | **ממשיכה בניווט האוטונומי המובנה** (אינה עוצרת) | סגול       |
| **ידני**       | **עוצרת וממתינה** עד שהקשר יתחדש                | אדום       |

(סף הניתוק: `FAILSAFE_TIMEOUT_MS`. באתחול הסירה במצב ידני-עצור עד שתגיע פקודה.)
