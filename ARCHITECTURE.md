# ארכיטקטורת המערכת — סירה אוטונומית

## סקירה כללית

המערכת מורכבת משלושה רכיבים עיקריים:

```
┌─────────────────────────────────────────────────────────┐
│                      מחשב (PC)                          │
│  - כל הלוגיקה (ניווט, שליטה ידנית, אוטונומי)           │
│  - ממשק משתמש (תצוגת מכ"ם, בקרה)                       │
└───────────────────────┬─────────────────────────────────┘
                        │ USB / Serial
                        ▼
┌─────────────────────────────────────────────────────────┐
│            ארדואינו חוף (shore) — ממסר                  │
│  - מקבל פקודות מהמחשב ומשדר לסירה (RF24)               │
│  - מקבל טלמטריה מהסירה ומעביר למחשב                    │
│  - אין לוגיקה — רק ממסר שקוף                           │
└───────────────────────┬─────────────────────────────────┘
                        │ RF24 (2.4GHz)
                        ▼
┌─────────────────────────────────────────────────────────┐
│            ארדואינו סירה (boat) — לקוח                  │
│  - מקבל פקודות ומפעיל מנועים (L298N)                   │
│  - קורא חיישני אולטרה-סוני (x4)                        │
│  - שולח טלמטריה חזרה לחוף                              │
│  - אין לוגיקה — מבצע פקודות בלבד                       │
└─────────────────────────────────────────────────────────┘
```

---

## עיקרון מרכזי: הלוגיקה כולה על המחשב

כל החלטה — אוטונומית או ידנית — מתקבלת על המחשב.  
הארדואינו‍ים משמשים כ**שכבת חומרה בלבד** (I/O Bridge).

**יתרון:** כל שינוי לוגי (אלגוריתם ניווט, ממשק משתמש, פרמטרים) מתבצע **במחשב בלבד ואינו דורש צריבה מחדש** על הארדואינו — כל עוד לא נוסף או הוסר רכיב חומרה.

## פרוטוקול תקשורת

### מחשב → ארדואינו חוף (Serial, 115200 baud)

```
<leftSpeed>,<rightSpeed>,<winchSpeed>,<radarAngle>\n
```

| שדה          | טווח        | תיאור                            |
| ------------ | ----------- | -------------------------------- |
| `leftSpeed`  | -255 עד 255 | מהירות מנוע שמאל (שלילי = אחורה) |
| `rightSpeed` | -255 עד 255 | מהירות מנוע ימין                 |
| `winchSpeed` | -255 עד 255 | מהירות כננת הרשת                 |
| `radarAngle` | 0 עד 180    | זווית סרוו המכ"ם                 |

### ארדואינו חוף → מחשב (Serial, 115200 baud)

```
<usRadar>,<usFront>,<usLeft>,<usRight>,<radarAngle>\n
```

| שדה       | יחידה | תיאור                     |
| --------- | ----- | ------------------------- |
| `usRadar` | ס"מ   | מרחק חיישן המכ"ם (מסתובב) |
| `usFront` | ס"מ   | מרחק חיישן קדמי קבוע      |
| `usLeft`  | ס"מ   | מרחק חיישן שמאלי קבוע     |
| `usRight` | ס"מ   | מרחק חיישן ימני קבוע      |
| `radarAngle` | מעלות | זווית הסרוו בזמן הדגימה |

## מצבי שליטה

### 1. שליטה ידנית (Manual)

- המשתמש שולט במנועים ישירות (מקלדת / ג'ויסטיק)
- תצוגת מכ"ם בזמן אמת על המסך (בנויה מנתוני `usRadar` + `radarAngle`)
- ניתן לשנות זווית מכ"ם ידנית

### 2. שליטה אוטונומית (Autonomous)

- המחשב בונה **תמונת מכ"ם** מנתוני `usRadar` תוך כדי סיבוב הסרוו
- אלגוריתם הניווט על המחשב מחליט על כיוון ומהירות
- הפקודות נשלחות אוטומטית לסירה
- המשתמש יכול להפריע / לעצור בכל עת

### מעבר בין מצבים

- **המחשב בלבד** שולט במעבר בין מצב ידני לאוטונומי
- ניתן לממש כמתג במשק בקרה, קיצור מקלדת, או פקודת תוכנה
- במקרה חירום: פקודת עצירה (`0,0,0,90`) נשלחת מיידית ללא תלות במצב
