# API — תקשורת מחשב ↔ ארדואינו חוף

מסמך זה מתאר את כל מה שתוכנת המחשב צריכה **לשלוח** ו**לקבל** דרך הפורט הטורי (Serial)
כדי לשלוט בסירה ולקבל ממנה נתונים.

---

## 1. חיבור (Connection)

| פרמטר        | ערך              |
| ------------ | ---------------- |
| ממשק         | USB / Serial     |
| Baud rate    | **115200**       |
| Data bits    | 8                |
| Parity       | None             |
| Stop bits    | 1                |
| Flow control | None             |
| קידוד תווים  | ASCII            |
| סיום שורה    | `\n` (LF, 0x0A)  |

> כל הפקודות וכל הטלמטריה הן **טקסט (ASCII)** בפורמט CSV, שורה אחת לכל הודעה,
> המסתיימת בתו שורה חדשה `\n`.

---

## 2. פקודות: מחשב → סירה (TX)

שולחים **שורה אחת** עם 4 ערכים מספריים שלמים, מופרדים בפסיקים, ומסתיימת ב-`\n`:

```
<leftSpeed>,<rightSpeed>,<winchSpeed>,<radarAngle>\n
```

**חובה לשלוח את כל 4 הערכים.** אם מספר השדות אינו 4 — הפקודה **תידחה** והסירה
תמשיך לפעול לפי הפקודה האחרונה התקינה.

### שדות הפקודה

| #   | שדה          | סוג | טווח חוקי   | ערך עצירה | תיאור                                            |
| --- | ------------ | --- | ----------- | --------- | ------------------------------------------------ |
| 1   | `leftSpeed`  | int | -255 עד 255 | `0`       | מהירות מנוע שמאל. חיובי = קדימה, שלילי = אחורה.   |
| 2   | `rightSpeed` | int | -255 עד 255 | `0`       | מהירות מנוע ימין. חיובי = קדימה, שלילי = אחורה.   |
| 3   | `winchSpeed` | int | -255 עד 255 | `0`       | מהירות כננת הרשת. חיובי / שלילי = כיוון גלגול.    |
| 4   | `radarAngle` | int | 0 עד 180    | `90`      | זווית סרוו המכ"ם במעלות. `90` = מרכז (פנים קדימה).|

> **הערות חשובות:**
> - הערכים `-255..255` תואמים ל-PWM של המנוע (`0` = עצור, `255` = מלא).
> - הסימן (חיובי/שלילי) קובע את כיוון הסיבוב; הגודל המוחלט קובע את העוצמה.
> - אין אצירה/Buffering בצד הסירה — כל פקודה חדשה **מחליפה** את הקודמת מיידית.
> - מומלץ לשלוח פקודות בקצב קבוע (למשל 10–20 פעם בשנייה) כדי לשמור על שליטה רציפה.

### דוגמאות לפקודות

| מטרה                                  | מה לשלוח           |
| ------------------------------------- | ------------------ |
| עצירה מלאה (גם חירום) + מכ"ם למרכז     | `0,0,0,90\n`       |
| קדימה ישר, מהירות בינונית              | `150,150,0,90\n`   |
| קדימה במהירות מלאה                     | `255,255,0,90\n`   |
| אחורה לאט                              | `-100,-100,0,90\n` |
| פנייה במקום ימינה (שמאל קדימה, ימין אחורה) | `150,-150,0,90\n`  |
| פנייה במקום שמאלה                      | `-150,150,0,90\n`  |
| סיבוב הסרוו של המכ"ם לזווית 45°        | `0,0,0,45\n`       |
| גלגול רשת פנימה תוך עמידה במקום        | `0,0,200,90\n`     |
| שחרור רשת (כיוון הפוך)                 | `0,0,-200,90\n`    |

## 3. טלמטריה: סירה → מחשב (RX)

הסירה משדרת טלמטריה אוטומטית בקצב של **כ-10 הרץ** (כל ~100ms).
כל הודעה היא **שורה אחת** עם 5 ערכים שלמים מופרדים בפסיקים, מסתיימת ב-`\n`:

```
<usRadar>,<usFront>,<usLeft>,<usRight>,<radarAngle>\n
```

### שדות הטלמטריה

| #   | שדה       | יחידה | תיאור                              |
| --- | --------- | ----- | ---------------------------------- |
| 1   | `usRadar` | ס"מ   | מרחק מחיישן המכ"ם המסתובב.          |
| 2   | `usFront` | ס"מ   | מרחק מהחיישן הקדמי הקבוע.           |
| 3   | `usLeft`  | ס"מ   | מרחק מהחיישן השמאלי הקבוע.          |
| 4   | `usRight` | ס"מ   | מרחק מהחיישן הימני הקבוע.           |
| 5   | `radarAngle` | מעלות | זווית הסרוו בזמן דגימת `usRadar` (0..180). |

> **ערך מיוחד:** הערך `999` משמעו **אין קליטה / מחוץ לטווח** (מעבר ל-~3.4 מטר).
> יש להתייחס אליו כ"שטח פנוי" ולא כמרחק אמיתי.

### בניית תמונת מכ"ם

הערך `usRadar` משויך לזווית הסרוו שנמדדה בפועל בשדה `radarAngle` שהתקבל בטלמטריה.
כדי לבנות סריקה, יש לשלב בצד המחשב:

```
זווית סריקה = radarAngle שהתקבל   →   מרחק = usRadar שהתקבל
```

על-ידי שינוי הדרגתי של `radarAngle` (0→180) ושמירת זוגות (זווית, מרחק)
ניתן לבנות מפת מכ"ם מלאה במחשב.

---

## 4. שגיאות

ייתכן והממסר ישלח שורות טקסט שאינן טלמטריה במקרה של תקלה:

| הודעה                                       | משמעות                               |
| ------------------------------------------- | ------------------------------------ |
| `ERROR: Radio hardware is not responding!`  | חומרת הרדיו בחוף תקולה / לא מחוברת.   |
| `ERROR: Radio transmission failed`          | השידור לסירה נכשל (טווח / הפרעה).     |

> תוכנת המחשב צריכה לזהות שורות שאינן בפורמט של 5 מספרים (כמו השורות לעיל)
> ולטפל בהן כהודעות שגיאה, לא כטלמטריה.

---

## 5. סדר עבודה מומלץ (Pseudo-flow)

```
1. פתח פורט טורי ב-115200 baud.
2. (אופציונלי) המתן ~2 שניות לאתחול הארדואינו.
3. בלולאת הבקרה (10–20Hz):
     a. חשב/קרא פקודה רצויה.
     b. שלח "leftSpeed,rightSpeed,winchSpeed,radarAngle\n".
4. בלולאת הקריאה (מקבילה):
     a. קרא שורה עד '\n'.
       b. אם 5 מספרים → עדכן טלמטריה (usRadar, usFront, usLeft, usRight, radarAngle).
        אחרת → טפל כהודעת שגיאה.
5. בעת יציאה / חירום: שלח "0,0,0,90\n" כדי לעצור הכל.
```

---

## 6. תקציר מהיר (Cheat Sheet)

```
שליחה (PC → Boat):   leftSpeed,rightSpeed,winchSpeed,radarAngle\n
                     מנועים: -255..255   |   סרוו מכ"ם: 0..180
                     עצירה:  0,0,0,90\n

קבלה  (Boat → PC):   usRadar,usFront,usLeft,usRight,radarAngle\n   (~10Hz)
                     יחידות: ס"מ   |   999 = אין קליטה/רחוק מדי
```
